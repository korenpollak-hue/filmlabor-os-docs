Aktive Dateien. Ein Ordner pro Client, Content-Drafts, Projekte, Research. Hier passiert die Arbeit.

Methodik

Jake Van Clief Prinzipien

Die Ordnerstruktur und Arbeitsweise folgen sechs klaren Regeln.

1

CONTEXT.md zuerst

Jeder Hauptordner hat eine CONTEXT.md. Lies sie bevor du arbeitest.

2

PROGRESS.md pro Client

Session-Start: lesen. Session-Ende: updaten. Kontinuitaet ueber Sessions.

3

Routing Table

CLAUDE.md definiert fuer jede Aufgabe den Pfad: welcher Skill, welcher Ordner.

4

Skills-First

Skill existiert? Folge ihm. Nicht vorhanden? Baue einen neuen.

5

USER / SYSTEM Trennung

context/ und workspace/ = User-Daten. code/ und scripts/ = System.

6

Quality Gates

Jeder Output wird durch Scoring und Review geprueft bevor er rausgeht.

Setup

Quick Start

In vier Schritten zum laufenden System.

Installation

1

Obsidian Vault oeffnen — vault/ als Obsidian Vault oeffnen

2

System-Skills installieren — cp -r claude-skills/* ~/.claude/skills/

3

Environment einrichten — env-example.txt nach .env kopieren, eigene API Keys eintragen

4

CLAUDE.md lesen — Dort steht die komplette Routing Table. Das ist dein Startpunkt.
