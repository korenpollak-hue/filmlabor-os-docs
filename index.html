Effizienz

Token-Optimierung

Das System spart 60-90% Tokens durch gezielte Section-Notation und Skip-Regeln.

✗ Ohne System

"Lies die komplette SKILL.md"
"Lies alle Context-Dateien"
"Lies alle Client-Dateien"

= Alles wird geladen
= Kontext voll nach 2 Tasks
~200K Tokens

✓ Mit FilmlaborOS

SKILL.md §Stages §Prinzipien
= Nur 2 Sections geladen

Skip-Regeln:
  Sales-Task? → learning/ ignorieren
  Content-Task? → scripts/ ignorieren
  Code-Task? → workspace/ ignorieren
~20K Tokens

Token-Verbrauch pro Methode

Komplette Datei laden100%
~50K
§-Notation (2-3 Sections)15%
~8K
Skip-Regeln aktiv5%
~3K
§-Notation Beispiel: SKILL.md §Stages §Prinzipien = Lade NUR die Sections "Stages" und "Prinzipien" = Rest wird uebersprungen SKILL.md (ohne §) = Gesamte Datei laden Skip-Regeln in CLAUDE.md: | Arbeit an | NICHT laden | |------------|--------------------------| | Sales | learning/, content/ | | Content | scripts/, api-config.md | | Code | workspace/, learning/ |
Foundation

Warum Obsidian?

Obsidian ist die lokale Wissensbasis. Alles liegt als Markdown auf deiner Festplatte — kein Cloud-Lock-in, kein Server.

Md

Plain Markdown

Alles sind .md Dateien. Claude Code liest und schreibt sie direkt. Kein proprietaeres Format.

[[

Wikilinks

[[Dateiname]] verlinkt Dateien sofort. Baut ein Wissensnetz zwischen Skills, Clients und Projekten.

G

Graph View

Visualisiert alle Verbindungen als interaktives Netzwerk. Zeigt wie Skills und Clients zusammenhaengen.

#

Tags & Properties

YAML-Frontmatter fuer Metadaten. Tags fuer schnelle Filterung. status: draft / review / final.

!

Embeds & Callouts

![[Datei]] bettet Content ein. Callouts fuer Warnungen, Tipps, wichtige Hinweise.

0

Lokal & Offline

Alles auf deiner Festplatte. Kein Account. Keine Cloud. Git fuer Sync und Backup.

Sicherheit

API-Keys & Credentials

Das System arbeitet mit vielen externen APIs. Hier die Sicherheitsregeln.

⚠ Niemals API-Keys in Dateien speichern!

Alle Secrets gehoeren in die .env Datei. Diese wird NIE ins Repository committed. CLAUDE.md und die Skills referenzieren nur Umgebungsvariablen, nie echte Keys.

✗ Falsch

Key direkt im Code:
api_key = "sk-abc123..."

Key in einer .md Datei:
Airtable Token: pat123...

Key als Default-Wert:
os.getenv("KEY", "sk-real")

✓ Richtig

Aus .env laden:
api_key = os.getenv("API_KEY")

In .env Datei:
AIRTABLE_API_TOKEN=pat123...

.env in .gitignore:
.env steht in .gitignore

✓ .env.example

Template mit leeren Werten. Wird ins Repo committed. Zeigt welche Keys benoetigt werden, ohne echte Werte.

✓ .credentials/

OAuth-Tokens fuer Google APIs. Wird beim ersten Lauf per Browser-Flow generiert. Ebenfalls in .gitignore.

Security Checklist

  • ✓
    .env nie committen — Steht in .gitignore. Immer pruefen vor git add.
  • ✓
    Keys rotieren — Wenn ein Key geleakt wurde, sofort in der API-Console rotieren.
  • ✓
    Least Privilege — API-Tokens nur mit den Rechten erstellen die noetig sind.
  • ✓
    Vor dem Teilen scannen — grep -r "sk-\|pat[A-Z]\|ghp_" . vor jedem Export.
  • ✓
    MCP Server pruefen — .mcp.json kann Tokens enthalten. Nie oeffentlich teilen.
  • ✓
    Keine Defaults mit echten Keys — os.getenv("KEY", "") statt os.getenv("KEY", "sk-real...")
