Routing: Von der Aufgabe zum Output

Du sagst was du willst. CLAUDE.md weiss welcher Skill, welcher Ordner, welches Script.

Skills

Wie ein Skill funktioniert

Jeder der 169 Skills folgt dem gleichen Pattern. SKILL.md definiert die Stages.

# angebot-pdf > Erstellt professionelle Angebote als PDF. ## Stages
1. Template aus examples/ laden
2. Client-Daten einsetzen
3. Spec-Review-Gate durchlaufen
4. PDF rendern
## Prinzipien - Minimum 3.500 EUR - Immer Upsell-Option - Zahlung: 50% / 50% ## Output workspace/docs/YYYY-MM-DD-client-angebot.pdf

Der AI-Agent durchlaeuft die Stages:

1

Template laden

Aus examples/ das passende Format waehlen

2

Client-Daten

Name, Projekt, Budget aus workspace/clients/

3

Review Gate

Qualitaetspruefung: Preis, Umfang, Formatierung

4

PDF rendern

Output nach workspace/docs/ mit Datums-Prefix

Jeder Skill hat: SKILL.md (Anleitung), examples/ (Referenz-Outputs), references/ (Externe Docs), scripts/ (Code)

Sicherheit

API-Keys & Credentials

Das System arbeitet mit externen APIs. Sicherheitsregeln beachten.

Niemals API-Keys in Dateien speichern.

Alle Secrets gehoeren in die .env Datei. Diese wird nie ins Repository committed.

✗ Falsch

api_key = "sk-abc123real..."
os.getenv("KEY", "sk-real-fallback")
# Token: pat123... in einer .md

✓ Richtig

api_key = os.getenv("API_KEY")
# .env Datei (in .gitignore):
AIRTABLE_API_TOKEN=pat123...
ANTHROPIC_API_KEY=sk-ant-...
  • ✓
    .env nie committen — Steht in .gitignore
  • ✓
    Keys sofort rotieren wenn sie geleakt wurden
  • ✓
    Least Privilege — Tokens nur mit noetigen Rechten
  • ✓
    Vor dem Teilen: grep -r "sk-\|pat[A-Z]\|ghp_" .
  • ✓
    .mcp.json nie teilen — kann Tokens enthalten
